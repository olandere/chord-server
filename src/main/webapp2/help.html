Chord charts - beta

The Chord field accepts a variety of inputs - Chord names (e.g. Am, GM9, F#m7b5), chord fingerings (x 0 2 2 2 0), comma separated lists of chord names or fingerings, and chord names with any of the commands below prefixed to them.

Commands

  • root: - Renders the chord with the Root note (R) as the lowest note.
  • drop2: - Renders the chord in Drop 2 voicing.
  • drop24: - Renders the chord in Drop 2 and 4 voicing.
  • shell: - Renders the chord as a shell chord (no 5, and only the highest extension).

Examples

  • Am7 - Renders all possible fingerings of the Am7 chord.
  • Am7@5 - Renders all possible fingerings of the Am7 chord with at least 1 note at the 5th fret.
  • drop2: Am7 - Renders all possible fingerings of the Am7 chord using Drop 2 voicing.
  • Am7, D7, GM7 - Renders the closest fingerings for the three chords making up this progression.
  • root: Am7, drop2: D7, shell: GM9 - Yes, you can get this specific!
  • 0 2 2 1 0 0 - Identifies most likely chord (E) and renders it.
  • 0 2 2 1 0 0, x 0 2 2 1 0 - Identifies and renders this list of chords (E and Am).
(NOTE: The code for the chord progressions currently just uses a distance measure between the chords, so it makes odd choices at times. Ideally it would take voice leading into account too so that they sound 'right'.)

Tunings

The Tuning field accepts tunings in two formats:
  • List of note names - EADGBE
  • Joni Mitchell notation - E55545
You are not restricted to guitar tunings. Banjo, ukulele and other string instruments are supported, just enter the appropriate tuning.
